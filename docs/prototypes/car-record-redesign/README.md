# 车辆保养 App 重新设计原型

## 产物

- `index.html`：可点击静态 HTML 原型，可直接用浏览器打开。

## 设计方向

- 工具型移动 App，不做营销页。
- iOS 与 Android 共用同一套信息架构、颜色、圆角、间距、按钮、表单、列表、底部导航和底部弹层样式。
- 主导航固定为“提醒 / 记录 / 我的”，新增保养记录作为跨模块高频浮动动作。
- 业务围绕“当前应用车辆”展开，提醒、记录和新增记录都明确跟随该上下文。

## 可点击路径

- 底部导航切换：提醒、记录、我的。
- 右下角加号：新增保养记录，保存后更新当前车辆里程。
- 首页车辆切换：打开车辆选择弹层并切换当前应用车辆。
- 保养项目管理：打开项目弹层，启用/禁用项目，新增自定义项目。
- 记录页：按周期/按项目切换、删除记录、移除单个项目。
- 我的页：添加车辆、导出备份、恢复数据、切换手动日期。

## 落地约束

- 日期统一按 `yyyy-MM-dd` 展示和输入。
- 默认项目用稳定 `catalogKey`，不要把展示名称作为主键。
- 同车同日唯一、单条记录内项目去重、车辆里程只增不减等规则需要在真实实现中继续保留。
- 未来原生落地时，可把本原型的 CSS token 转成 iOS/Android 设计 token。
